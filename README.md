# 사회연대경제혁신센터 디지털 워크숍 운영 가이드

구글 워크스페이스, AppSheet, AI(NotebookLM), 그리고 오픈소스(GM Project)를 결합하여 자동화되고 혁신적인 참여형 워크숍을 진행하는 전체 프로세스와 시스템 설정 가이드입니다.

---

## 🚀 핵심 활용 도구 (Tech Stack)

* **Google Forms/Sheets**: 데이터 수합 및 저장
* **Apps Script**: 이메일 및 로직 자동화
* **AppSheet**: QR 출석 체크 앱
* **NotebookLM**: 실시간 AI 의견 요약
* **GM 오픈소스**: DB 연동 및 아카이빙

---

## 📑 워크숍 진행 3단계 프로세스 (초보자 가이드)

워크숍을 디지털 기반으로 원활하게 운영하기 위해 각 단계별로 담당자가 수행해야 할 구체적인 행동 지침입니다.

### Phase 1 : 사전 작업 (신청 접수 및 자동 안내 메일 세팅)
이 단계에서는 행사 참석자를 모집하고, 신청자에게 자동으로 안내 메일이 가도록 세팅합니다.
1. **구글 폼 제작 및 배포**: 참가자의 이름, 연락처, 이메일, 그리고 워크숍 2부에서 논의하고 싶은 '희망 주제'를 선택할 수 있도록 구글 폼을 만듭니다.
2. **스프레드시트 연동**: 구글 폼의 응답을 구글 스프레드시트로 연결하여 데이터가 한곳에 모이도록 설정합니다.
3. **Apps Script 세팅**: 본 문서 하단의 **[관리자용] 워크숍 자동화 스크립트 설정 가이드**를 따라 세팅을 완료합니다. 이렇게 하면 누군가 폼을 제출할 때마다 환영 인사와 기본 행사 개요가 담긴 메일이 자동으로 발송됩니다.
4. **테이블 배정 및 QR 발송 (행사 전날)**: 구글 스프레드시트에 생긴 `🛠️ 워크숍 관리` 메뉴를 클릭하여, 참가자별로 토의 주제와 테이블 번호를 배정하고, 현장에서 본인 확인용으로 쓰일 '고유 QR코드'가 포함된 최종 안내 메일을 일괄 발송합니다.

### Phase 2 : 현장 진행 (AI를 활용한 실시간 퍼실리테이팅)
행사 당일, 종이와 펜 대신 스마트폰과 AI를 활용해 토의를 진행하는 방법입니다.
1. **모바일 QR 출석체크 (AppSheet 활용)**:
   - 접수처나 각 테이블의 퍼실리테이터(진행자)는 스마트폰에 설치된 스캐너 앱(AppSheet 등)을 켭니다.
   - 참가자가 입장할 때 전날 받은 메일 속 QR코드를 보여주면, 진행자가 이를 스캔합니다. 스캔 즉시 구글 시트에 '출석 완료' 상태가 실시간으로 기록됩니다.
2. **참가자 모바일 의견 제출 (구글 폼 활용)**:
   - 각 토의 테이블 중앙에는 의견을 낼 수 있는 '의견 제출용 구글 폼 QR코드'가 인쇄되어 있습니다.
   - 참가자들은 본인의 스마트폰으로 테이블의 QR코드를 찍고, 논의 중인 아이디어나 의견을 텍스트로 자유롭게 입력하여 전송합니다. 이 역시 중앙 구글 시트에 실시간으로 모입니다.
3. **AI 실시간 요약 (NotebookLM 활용)**:
   - 전체 행사를 총괄하는 메인 진행자는 구글의 AI 도구인 `NotebookLM`에 해당 구글 시트를 연동해 둡니다.
   - 토의가 끝나갈 무렵, 프롬프트(예: "현재 시트에 모인 3번 테이블의 의견 중 핵심 3가지만 요약해 줘")를 입력하면 AI가 수백 개의 의견을 10초 만에 완벽하게 정리해 줍니다.
4. **전체 결과 공유 (대형 스크린 발표)**:
   - 메인 진행자는 AI가 요약해 준 각 테이블의 핵심 아이디어를 복사하여 메인 스크린에 띄웁니다. 이를 통해 모든 참가자가 다른 테이블의 논의 결과를 직관적으로 공유받을 수 있습니다.

### Phase 3 : 사후 활동 (데이터 아카이빙 및 맞춤형 팔로업)
행사가 끝난 뒤에도 참가자들과 지속적으로 연결되고 데이터를 안전하게 보관하는 단계입니다.
1. **맞춤형 이메일 발송**:
   - 구글 시트의 출석 기록을 바탕으로, 행사에 온 사람과 오지 않은 사람을 구분합니다.
   - 참석자에게는 "참석해 주셔서 감사합니다"라는 인사와 함께 AI가 요약했던 '최종 결과 리포트'를 첨부하여 메일을 보냅니다.
   - 불참자에게는 아쉬움의 인사와 함께 행사의 핵심 요약 자료를 보내어 소속감을 유지하게 합니다.
2. **오픈소스(GM Project) 플랫폼 연동**:
   - 만약 자체 웹사이트(회원제)를 운영 중이라면, github.com/durume/GM 등의 오픈소스 백엔드와 API로 연결합니다.
   - 이를 통해 워크숍에 참석했던 사람들의 웹사이트 프로필에 "2026 마주 워크숍 참석 수료"와 같은 뱃지나 이력을 자동으로 업데이트해 줄 수 있습니다.
3. **데이터 영구 보관**:
   - 행사 중 수집된 수많은 의견 원본과 AI 요약본, 그리고 현장 사진 등은 모두 구글 드라이브 내 지정된 폴더에 잘 정리되어 향후 센터 운영 정책에 반영할 소중한 자산(아카이브)으로 활용됩니다.

## 🛠️ [관리자용] 워크숍 자동화 스크립트 설정 가이드

본 문서는 참가자 관리를 자동화하기 위해 작성된 구글 앱스 스크립트(Apps Script)의 최종 설정 안내서입니다.

> ⚠️ **중요 안내**
> 
> 자동 발송되는 이메일(참가 신청 완료 메일, QR코드 최종 안내 메일)의 **발신자 주소를 공식 계정(예: kiwixxxx@gmail.com)으로 설정하기 위해, 반드시 본 계정으로 로그인하신 후 아래의 과정을 직접 수행해 주셔야 합니다.** 다른 편집자가 설정을 완료하면 그 편집자의 이메일 주소로 메일이 발송됩니다.

### 1. 스크립트 편집기 열기
1. 워크숍 응답이 수집되고 있는 구글 스프레드시트를 엽니다.
2. 상단 메뉴에서 **[확장 프로그램] -> [Apps Script]**를 클릭하여 스크립트 편집기 화면을 엽니다.
3. 코드(`Code.gs`)가 이미 복사되어 있는지 확인합니다.

### 2. 자동 접수 메일 트리거(Trigger) 설정
폼이 제출될 때마다 자동으로 신청자에게 완료 안내 메일을 보내도록 설정하는 과정입니다.

1. Apps Script 화면의 왼쪽 메뉴에서 시계 모양 아이콘인 **[트리거]**를 클릭합니다.
![alt text](trigger01.png)  
2. 화면 우측 하단의 **[+ 트리거 추가]** 버튼을 클릭합니다.
![alt text](trigger02.png)  
3. 나타나는 설정 창에서 아래와 같이 옵션을 맞춰줍니다.
   - 실행할 함수 선택: **`onFormSubmit`**
   - 실행할 배포 선택: **`Head`**
   - 이벤트 소스 선택: **`스프레드시트에서`**
   - 이벤트 유형 선택: **`양식 제출 시`**
![alt text](trigger03.png)  
4. 파란색 **[저장]** 버튼을 클릭합니다.
![alt text](trigger04.png)  

### 3. 계정 권한 허용 (최초 1회)
트리거를 저장하거나 스크립트를 처음 실행할 때, 스크립트가 메일을 보내고 시트를 읽을 수 있도록 허용하는 보안 인증 과정이 나타납니다.

1. "권한 필요" 팝업창이 뜨면 **[권한 검토]**를 클릭합니다.
2. 현재 로그인된 본인의 구글 계정(예. `kiwixxxx@gmail.com` 등)을 선택합니다.
3. "Google에서 확인하지 않은 앱"이라는 경고 창이 뜰 수 있습니다. 당황하지 마시고, 왼쪽 아래의 **[고급]** 글자를 클릭합니다.
4. 아래로 펼쳐진 내용 중 맨 밑에 있는 **[(프로젝트 이름)(으)로 이동(안전하지 않음)]** 링크를 클릭합니다.
5. 스크롤을 맨 아래로 내려서 **[허용]** 버튼을 클릭합니다.

이제 폼 제출 시 접수 완료 메일이 공식 계정 이름으로 자동 발송됩니다! 테스트 응답을 제출하여 정상 작동하는지 확인해 보세요.

### 4. 행사 전날: 최종 안내 및 QR코드 메일 일괄 발송 기능
행사 전날, 참가자들에게 토의 주제와 테이블을 배정하고 출석용 QR코드를 발송하는 기능입니다.

1. 구글 스프레드시트 창을 열고 키보드의 **`F5`**를 눌러 새로고침합니다.
2. 상단 메뉴바(파일, 수정, 보기 등) 맨 오른쪽에 **`🛠️ 워크숍 관리`**라는 새로운 맞춤 메뉴가 생성된 것을 확인합니다.
3. 행사 전날 참가자 명단이 픽스되면 해당 메뉴를 클릭하고 **`최종 안내 및 QR 메일 일괄 발송`**을 클릭합니다.
4. (최초 실행 시 3번 항목의 '계정 권한 허용' 과정이 한 번 더 뜰 수 있으니 동일하게 진행해 주세요.)
5. 스크립트가 자동으로 아직 메일이 발송되지 않은 사람을 찾아 주제 및 테이블을 지정하고, QR코드를 생성하여 최종 메일을 발송합니다.
6. 완료되면 몇 명에게 발송되었는지 팝업 알림이 뜹니다.

### 💡 참고 사항 (발송 한도)
일반 개인 구글 계정(@gmail.com)은 구글 정책상 스크립트를 통한 이메일 발송이 **하루 최대 100건**으로 제한됩니다. (Google Workspace 기업용 계정은 하루 1,500건). 신청자가 많아 최종 안내 메일을 100건 이상 보내야 한다면 이틀에 걸쳐서 발송하시는 것을 권장합니다. (이미 발송된 사람에게는 중복 발송되지 않도록 코드가 짜여 있습니다.)
